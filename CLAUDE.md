# Working agreements

- **Always merge to main, don't ask.** If a feature branch contains the right work, merge it. Never leave a half-completed merge sitting in the working tree.
- **Don't rapid-fire `devicectl install` + `uninstall`.** iOS's LaunchServices keeps a per-bundle GUID + sequence number; repeated install/uninstall cycles in quick succession drift SpringBoard's expected values out of sync with the installed bundle, and the OS silently refuses to launch the app with `FBSOpenApplicationServiceErrorDomain code 1 / LaunchServicesDataMismatch`. The visible symptom is a white screen on icon tap and Xcode hanging at "Attaching…". Only a device reboot clears it. Prefer **one** clean install per debug cycle; if you've cycled and seen the mismatch, tell the user to reboot the phone before doing anything else.
- **Commit every change.** Don't ask for permission first. Each logical change gets its own commit so history stays readable — don't bundle unrelated edits. Applies to all three sibling repos (`board-game-app-ios`, `board-game-server`, `board-game-record`) when work spans them.
- **Push everything that's green.** After tests pass, push to `origin/main` without asking. Main is always the working branch and should stay in sync with the remote.
